## Motivation

### For what purpose was the dataset created?

This dataset was created to document the query history and function evaluations for an 8-week black-box optimization (BBO) challenge in the Imperial College ML/AI Capstone course. The dataset supports:

1. **Optimization Strategy Development**: Training surrogate models to predict function behavior in high-dimensional spaces
2. **Performance Analysis**: Tracking optimization trajectory across 8 unknown functions under strict budget constraints
3. **Research Transparency**: Enabling reproducibility of Bayesian optimization experiments in data-scarce environments
4. **Educational Assessment**: Demonstrating decision-making under uncertainty with N<20 samples per function

### Who created the dataset and on whose behalf?

**Created by**: Imperial ML/AI Capstone student (2025-2026 cohort)  
**On behalf of**: Academic research and course assessment requirements  
**Supervision**: Imperial College London faculty

### Who funded the creation of the dataset?

This dataset was created as part of coursework for the Imperial College ML/AI Capstone program. No external funding was involved.

## Composition

### What do the instances represent?

Each instance represents a **query-evaluation pair**:
- **Query**: Input vector in [0,1]^d (d = dimensionality of function)
- **Evaluation**: Scalar output value returned by black-box function
- **Metadata**: Week number, function ID, query index, strategy used

### How many instances are there?

**Total instances**: 82 query-evaluation pairs
- **Weeks 1-8**: 8 queries per function × 8 functions = 64 pairs
- **Week 9**: 2 queries (F4, F8 only)
- **Pre-collected data**: 175 samples provided by course (not generated by this project)
- **Week 10**: No new queries (budget exhausted)

### What data does each instance consist of?

```python
instance = {
    'function_id': int,           # 1-8
    'week': int,                  # 1-10
    'query_index': int,           # Sequential counter
    'input': np.ndarray,          # Shape (d,), dtype float64, range [0,1]
    'output': float,              # Scalar evaluation result
    'dimensionality': int,        # 2, 3, 4, 5, 6, or 8
    'strategy': str,              # E.g., "random", "EI", "UCB", "bounded_random"
    'timestamp': datetime,        # Submission timestamp
    'ensemble_prediction': float, # Surrogate model prediction (W3+)
    'acquisition_score': float,   # EI or UCB value (W3+)
}
```

## Collection Process

### How was the data collected?

**Query Generation Pipeline** (Weeks 1-9):

1. **Week 1 (Random Baseline)**:
   - Strategy: Uniform random sampling in [0,1]^d
   - Purpose: Establish baseline, explore function variability
   - No surrogate models used

2. **Week 2 (Boundary Exploration)**:
   - Strategy: Targeted boundary vs. interior sampling
   - Purpose: Differential diagnosis of function preferences
   - No surrogate models used

3. **Weeks 3-7 (Bayesian Optimization)**:
   - Surrogates: Gradient Boosting, Random Forest, SVM RBF, Gaussian Process
   - Acquisition: Expected Improvement (EI) and Upper Confidence Bound (UCB)
   - Training data: 175 pre-collected samples + cumulative weekly results
   - Strategy: Function-specific β parameters in UCB (0.5-5.0 range)

4. **Week 8 (Post-Collapse Recovery)**:
   - F5 collapse analysis (−88% from W7)
   - F4 breakthrough validation (+69%)
   - F2 chaos recovery attempt
   - Ensemble model refinement

5. **Week 9 (Strategic Allocation)**:
   - Budget constraint: 2 queries remaining (10 total limit)
   - Selection: F4 (breakthrough trajectory) and F8 (stability baseline)
   - Abandoned: 6 functions (F1, F2, F3, F5, F6, F7)

### Who was involved in the data collection?

**Single author**: All queries generated by one student as part of individual capstone project.

**No crowdsourcing**: Dataset entirely generated through algorithmic methods.

### Over what timeframe was the data collected?

**Duration**: 9 weeks (late January 2026 - mid-March 2026)  
**Cadence**: 1 submission per week (8 queries per submission in W1-W8, 2 queries in W9)  
**Week 10**: Final standings observation (no new data collection)

### Were any ethical review processes conducted?

**Not applicable**: Dataset contains only synthetic mathematical function evaluations with no human subjects or sensitive data. Standard academic integrity policies apply.

## Preprocessing, Cleaning, and Labeling

### Was any preprocessing applied?

**Minimal preprocessing**:

1. **Input Validation**:
   - Clipping to [0,1] bounds
   - Precision standardization (6 decimal places)
   - Dimensionality verification

2. **Output Recording**:
   - Raw values stored without normalization
   - No outlier removal (all values valid by definition)

3. **Feature Engineering** (for surrogate models):
   - StandardScaler normalization for model inputs
   - Polynomial features tested (Weeks 2-3)
   - PCA dimensionality reduction tested (Week 5)

### Was the "raw" data saved in addition to preprocessed data?

**Yes**: All raw query-evaluation pairs stored in NumPy arrays:
- `submissions/week_XX/queries.py` contains original input arrays
- Weekly result values documented in README.md
- No intermediate transformations applied to evaluation data

## Uses

### What tasks has the dataset been used for?

1. **Surrogate Model Training**: Gradient Boosting, Random Forest, SVM, Gaussian Process regressors
2. **Acquisition Function Optimization**: EI and UCB for query point selection
3. **Strategy Evaluation**: Comparing random, boundary, exploitation, exploration approaches
4. **Volatility Analysis**: Identifying chaotic (F2), stable (F8), and collapsing (F5) functions
5. **Portfolio Risk Management**: Allocating limited budget under uncertainty

### What tasks is the dataset suitable for?

**Appropriate uses**:
- Bayesian optimization research with N<20 samples
- Surrogate model benchmarking in data-scarce regimes
- Decision-making under uncertainty case studies
- Non-stationary function optimization research
- Educational demonstrations of exploration-exploitation trade-offs

**Limitations**:
- **Small sample size**: N=9-10 per function insufficient for deep learning
- **Unknown ground truth**: Black-box constraint prevents validation of surrogate accuracy
- **Limited generalization**: Results specific to these 8 functions
- **Temporal confounding**: Weekly submission cadence may introduce time-based biases

### What tasks is the dataset NOT suitable for?

**Inappropriate uses**:
- Large-scale machine learning model training (N too small)
- Causal inference (no controlled experiments)
- Transfer learning to unrelated optimization domains
- Real-world safety-critical applications (insufficient validation)
- Benchmarking "optimal" strategies (ground truth unknown)
